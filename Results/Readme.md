# Results and Analysis Outputs

## Overview

This folder contains the output artifacts generated by the machine learning and drift analysis pipeline. These files represent the final results of the system, including drift detection, behavioral analysis, and model performance evaluation.

The purpose of this folder is to provide **evidence of system behavior under drift conditions** and enable reproducibility and validation of results.

---

## File Structure

```bash
.
├── behavior_deltas.txt
├── drift_detection_results.json
├── model_degradation_report.txt
```

---

## 1. behavior_deltas.txt

### Purpose

This file contains the results of behavioral analysis using Markov Chains. It quantifies how user navigation patterns change due to drift.

### Description

The system computes transition probabilities between different user states (pages) and compares them between:

* Baseline (non-drifted data)
* Drifted data

The key metric used is:

[
\Delta P = |P_{baseline} - P_{drifted}|
]

### Example Output

* Baseline transition probability (Product → Cart): 0.6620
* Drifted transition probability (Product → Cart): 0.0293
* Delta P: 0.6326

### Interpretation

A high Delta P indicates a significant behavioral shift. In this case, users are far less likely to add products to the cart after drift, demonstrating strong behavioral impact.

---

## 2. drift_detection_results.json

### Purpose

This file stores the drift detection results generated using the ADWIN algorithm.

### Description

The system monitors two data streams:

* Feature stream (smoothed price)
* Behavioral stream (rolling conversion rate)

Drift is detected when statistically significant changes occur in either stream.

### Structure

```json
{
  "price_drift_indices": [...],
  "behavior_drift_indices": [...],
  "combined_first_detection": ...
}
```

### Key Fields

* price_drift_indices: List of row indices where feature drift was detected
* behavior_drift_indices: List of row indices where behavioral drift was detected
* combined_first_detection: The earliest detected drift point

### Interpretation

* Early detection indicates sensitivity to distribution changes
* Multiple detections indicate sustained or evolving drift
* Comparison with ground truth allows validation of detection accuracy

---

## 3. model_degradation_report.txt

### Purpose

This file contains the results of machine learning model evaluation under drift conditions.

### Description

An XGBoost model is trained on baseline (clean) data and evaluated on both:

* Validation set (clean data)
* Drifted data

### Metrics Reported

* Validation Accuracy (baseline performance)
* Drifted Accuracy (performance under drift)
* Accuracy Collapse (difference between the two)

### Example Output

* Validation Accuracy: 0.9111
* Drifted Accuracy: 0.7213
* Accuracy Collapse: 0.1898

### Interpretation

A significant drop in accuracy indicates that the model fails to generalize when the data distribution changes. This highlights the impact of concept drift on static machine learning models.

## Relationship Between Files

These files together provide a complete picture of system behavior:

* drift_detection_results.json identifies when drift occurs
* behavior_deltas.txt explains how user behavior changes
* model_degradation_report.txt shows how model performance is affected
